# 🎧 Loca

**Loca** — це концепт інноваційного мобільного музичного стрімінгового сервісу, головною метою якого є підтримка локальних артистів та розвиток регіональних музичних спільнот. Платформа проєктується для поєднання функціоналу фонового прослуховування з інструментами геолокаційного пошуку виконавців та формування емоційного зв'язку між слухачами.

## 🌟 Ключові особливості

* 📍 **Музика поруч (Local Artists)**: Ядро платформи. Передбачає використання геопросторових запитів для пошуку та популяризації виконавців у безпосередньому радіусі перебування користувача. Розділ функціонуватиме як цифровий майданчик для просування незалежних локальних музикантів.
* 📝 **Стрічка спогадів**: Інтеграція текстових нотаток безпосередньо до аудіотреків. Користувачі матимуть можливість анонімно залишати особисті історії та переглядати асоціації інших людей, пов'язані з конкретною композицією.
* 🎧 **Синхронний стрім**: Механізм групового прослуховування у віртуальних кімнатах. Забезпечуватиме одночасне відтворення аудіо з повною двосторонньою синхронізацією команд керування (пауза, перемикання) між пристроями всіх учасників.
* 📖 **Легенди пісень**: Інформаційний розділ, що розкриватиме історичний та емоційний контекст створення треку, базуючись на даних від авторів.
* 🎲 **Абсолютний рандом**: Алгоритм примусового вибору випадкової композиції з глобальної бази для виходу з алгоритмічної бульбашки рекомендацій.
* 🤝 **Спільні плейлисти**: Функціонал колаборативного управління списками відтворення для користувачів платформи.

## 🛠 Запланований технологічний стек

**Клієнтська частина (Мобільний додаток):**
* **React Native (TypeScript)** — фреймворк для кросплатформної розробки мобільного застосунку під операційні системи iOS та Android.
* **react-native-track-player** — модуль для забезпечення стабільного фонового відтворення та обробки HTTP Range Requests.

**Серверна частина (Бекенд):**
* **ASP.NET Core Web API** — базовий фреймворк для обробки REST-запитів та реалізації бізнес-логіки.
* **SignalR** — протокол на базі WebSockets для маршрутизації подій реального часу (синхронізація відтворення у кімнатах).

**Зберігання даних та інфраструктура:**
* **PostgreSQL** — основна реляційна СУБД для роботи з профілями, плейлистами та текстовими спогадами.
* **PostGIS** — геопросторове розширення для індексації та математичного розрахунку відстаней функції «Музика поруч».
* **Redis** — in-memory кеш для зберігання токенів сесій та підтримки тимчасових станів активних кімнат.
* **MinIO** — S3-сумісне об'єктне сховище для фізичного збереження бінарних медіафайлів. Доступ до аудіо забезпечуватиметься через генерацію тимчасових посилань (Presigned URLs).
* **Docker** — контейнеризація інфраструктурних сервісів для ізольованого розгортання та локального тестування команди.

## 🔐 Аутентифікація
Проєктується на базі протоколів **OAuth 2.0** та **OpenID Connect**. Цей механізм забезпечить безпечний вхід у систему через сторонніх провайдерів ідентифікації без необхідності збереження паролів користувачів у внутрішній базі даних платформи.
